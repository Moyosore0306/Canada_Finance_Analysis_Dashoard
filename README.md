# 📊 Canada Finance Analysis Dashboard

## 📌 Project Overview
This project analyzes financial transactions and fraud patterns using Power BI. The goal is to provide insights into transaction behavior, detect fraud trends, and support data-driven decision-making in the banking sector.

---

## 📊 Dashboard Features

- Total Volume, Transactions, and Average Transaction KPIs
- Net Flow analysis (Credit vs Debit)
- Monthly transaction trends (April–March financial year)
- Merchant category spending analysis
- Account type performance
- Fraud detection and fraud rate analysis
- Fraud trends over time
- Fraud distribution by province and payment method

---

## 🧠 Key Insights

- Fraud rate is approximately 2%, indicating moderate risk exposure
- Fraud spikes in certain months suggest possible seasonal patterns
- Higher fraud concentration observed in provinces like British Columbia and Quebec
- Wire and transfer payment methods show higher fraud vulnerability
- Net flow fluctuations indicate periods where spending exceeds inflows

---

## 💡 Recommendations

- Implement real-time fraud monitoring systems for high-risk transactions
- Strengthen authentication (e.g., multi-factor authentication) for wire and transfer payments
- Apply region-specific fraud controls in high-risk provinces
- Increase monitoring during peak fraud months
- Provide customers with spending insights and budgeting tools
- Monitor accounts with consistent negative net flow to reduce financial risk

---

## 🏁 Conclusion

This analysis highlights the importance of proactive fraud detection and financial monitoring. While transaction activity is increasing, rising fraud cases and fluctuating net flows indicate the need for stronger risk management strategies.

By leveraging data analytics tools such as Power BI, financial institutions can gain actionable insights, improve security measures, and enhance overall decision-making.
